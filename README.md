Description

This project is built as a practical training exercise for students to learn real-world teamwork, GitHub workflow, task management, and collaborative software development.

Each team member is responsible for a specific part of the system, and all work is integrated through Pull Requests, Issue tracking, and code review.

    Modular architecture
    Separated tasks by contributors
    Clean Git workflow
    Review-based merging
    Issue-driven development

Team Workflow

    Every member works inside their own branch
    Each feature is submitted as a Pull Request
    Code review by at least one teammate
    Merge only after approval
    All tasks are created & tracked in Issues
    Milestones define the project timeline
